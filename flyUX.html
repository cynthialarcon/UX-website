Flight booking system

Client: Fly UX


THE PROBLEM

Fly UX is a startup airline company that is looking to provide the user with a stress-free booking system.

Fly UX is a startup airline company that is looking to use great design as a source of competitive advantage over bigger but also much more bureaucratic airlines. Other airlines are failing at providing a great user experience, Fly UX would have a better design that will help to grab market share.

The aim of this project is to focus on the flight booking process: How users can search, find, and select flights on desktop.

This is the case study for my final project for the Diploma in UX Design by UX Design Institue. The academic goal was to understand the UX design process, heuristics, design/ build a prototype and create a detailed set of accompanying wireframes. Everything I’d need to do to start working with a team of developers to get software built.



MY RESPONSABILITIES

  • Competitive benchmark
  • Online survey
  • Note taking
  • Usability test
  • Afﬁnity diagram
  • Customer journey map
  • Flow diagram
  • Interaction design
  • Prototype
  • Usability test for prototype

MAIN GOAL

Design a user-oriented solution that would target pain points seen in competitors websites and eases the process for the user.

TIMELINE AND DELIVERABLES

Four months:
  • Research: Includes Competitive benchmark, Online survey, Note taking, Usability test
  • Analysis: Afﬁnity diagram and Customer journey map
  • Design: Flow diagram and Interaction design
  • Prototyping and Wireframing: Prototype, Usability test for prototype and Wireframes

Research

The research stage involved gathering information about competitors and users. Also learning research techniques like interviews, creating surveys, executed usability tests and practicing note-taking.

This stage was completed in one month. Below, it is described each of the deliverables.

Competitive benchmark

Analyzed 4 airline competitors with the goal to get a list of good practices,
negative points to avoid, and star points that would enhance the experience.
Description

Analyzed 4 airline competitors, focused on the home page, flight search, booking process, and changes to the booking. The competitors were Etihad, Sinapore Airlines, Emirates, and Ryanair. The outcome of this was a list of conclusions per page on good practices, negative points to avoid, and star points that would enhance the experience.

Objective
  1. To analyze direct competitors
  2. To know how best-in-class websites solve the booking process.
  3. To know what are they doing really well.
  4. To know what can be improved.
  5. To find established conventions that I needed to follow.

Survey results

Objective:

Learn more about the goals of people that use airline websites and apps — what are they trying to do, what (if anything) is preventing them from doing it, what else would they like to be able to do on airline websites. Learn how to create an online survey.

Analysis

The analysis stage involved reviewing the competitive benchmarking document, survey results, and usability test recording. Also learning research techniques like affinity diagram and user journey map.

This stage was completed in one month. Below, it is described each of the deliverables.

Afﬁnity diagram

Reviewed competitive benchmarking document, survey results, and
usability test recording to put structure on qualitative research data.
Description

Reviewed the data gathered from the research phase to describe the current user experience: goals, behaviours, pain points, mental models and contextual information.

Objective
  1. To learn how to put structure on qualitative research data.
  2. To learn the power of analysis.
  3. To organize data to create a customer journey map.
Process
  1. Using Post-it notes, wrote down notes and observations from my notes to articulate the current airline booking experience.
  2. Wrote one observation per Post-it as descriptive as possible.
  3. Sticked them all on a wall — brainstorm
  4. Organized them into groups that make sense
The final groups I collected were:
  1. Summary
  2. Progress bar/ navigation bars
  3. Behaviours before choosing an airline website
  4. Contextural information
  5. Search of the flights
  6. Goals
  7. Search area
  8. Pain points
  9. Mental models
  10. Home page
  11. UI
  12. Calendar
  13. Long list of flight results
  14. Usability
  15. Provide available flights /prices in other days
  16. Behaviours on airline websites
  17. Flight details
  18. Adding passengers
  19. Payment
  20. One page Flow
  21. Wording of buttons and numbers
  22. Sharing information before booking
  23. Results page
  24. Other (needed more info)

User journey map

Created a user journey map highlighting the goals, behaviours,
context, mental model, pain point and wishes at every stage.
Description

Created a user journey map highlighting the goals, behaviours, context, mental model, pain point and wishes at every stage. Also, the user experience, whether positive, negative, or neutral.

Objective
  1. To learn how to translate research data into a structured document.
  2. To build on the work from previous research.
  3. To analyse data from raw research.

The columns highlighted in orange are part of a loop that the user executes n times until they decide to select a flight to then purchase it.

The beauty of this journey map is to clearly see what are the pain points at which stage to then target them.

Design

The design phase involved taking all the previous information and analysis to make informed decisions and sketch/design the interaction for Fly UX software.

This stage was completed in one month. Below, it is described each of the deliverables.

Flow diagram

Sketched the flow diagram from searching for flights
to selecting seats and right through to payment and confirmation.
Description

Sketched the flow diagram by hand before working on the final document. Each screen or state is represented by a box. Each interaction, such as choosing a departure airport or entering a date, is represented by a circle. Sketched the flow from searching for flights to selecting seats and right through to payment and confirmation.

Objective
  1. To define the high-level flow for booking flights on the new airline website.
  2. To address all the issues highlighted in the customer journey map.
  3. To prepare for designing screen details.

Interation design

Sketched the interaction design with the ﬂow already defined,
and having identiﬁed the key issues that needed to be addressed in the journey map .
Description

With the ﬂow and navigation already defined, and having identiﬁed the key issues that needed to be addressed in the journey map was time to create the interaction design.

Objective
  1. To sketch the screens and screen states for users ﬂowing through the ﬂight booking application.
  2. To address all the issues and user goals identiﬁed in the research and analysis.
  3. To build on the ﬂow diagram I created above.
  4. To learn how to use sketching as a tool for problem-solving and as a vital step before prototyping or digital wireframing.
Process
  1. Created a list of screens to design based on the ﬂow diagram. Started from the homepage of the website right through to the seat selection, payment and confirmation.
  2. Included screen state, when the state differs signiﬁcantly based on users actions.
  3. Iterated until happy with the results.
  4. At the end, I created gifs to show the interaction.
